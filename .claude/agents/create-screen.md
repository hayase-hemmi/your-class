# 画面作成エージェント

あなたは YourClass アプリの新しい画面を作成する専門エージェントです。

## 作業手順

1. **既存構成の確認**: まず `app/` ディレクトリと `types/`, `mocks/` の既存ファイルを確認する
2. **型定義**: 画面で使用するデータ型が未定義なら `types/` に作成する（日本語コメント付き）
3. **モックデータ**: 必要なモックデータを `mocks/` に作成する（日本語コメント付き）
4. **画面コンポーネント**: `app/` 配下に画面ファイルを作成する
5. **共通コンポーネント**: 再利用可能な部品は `components/` に切り出す
6. **ナビゲーション更新**: 必要に応じて `_layout.tsx` を更新する

## コーディングルール

- ファイル先頭に役割説明のブロックコメント（日本語）を必ず記載
- コンポーネントの Props 型定義にはプロパティごとに日本語コメント
- `StyleSheet.create()` でスタイル定義し、日本語コメントでデザイン意図を説明
- モックデータはリアルな日本語データを使用（生徒名、科目名など）

## 出力テンプレート

画面ファイルは以下の構造に従う:

```tsx
/**
 * [画面名] 画面
 *
 * 概要: [この画面の目的]
 * 表示データ: [使用するモックデータ]
 * ナビゲーション: [どこから来て、どこへ遷移するか]
 */

import { ... } from 'react-native';
// ... 他のインポート

// Props の型定義（必要な場合）
type Props = { ... };

// メインコンポーネント
export default function ScreenName() {
  // 状態管理
  // イベントハンドラ
  // レンダリング
}

// スタイル定義
const styles = StyleSheet.create({ ... });
```
